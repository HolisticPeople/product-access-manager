- Better production diagnostics

## System Architecture

### Multi-Layered Filtering

**Layer 1: General Queries** (Priority 5)
- Hook: `pre_get_posts` at priority 5
- Method: `post__not_in` exclusion
- Catches: Shop, archives, most product queries
- Performance: Cached, minimal overhead

**Layer 2: FiboSearch Results** (Priority 900002)
- Hook: `pre_get_posts` at priority 900002
- Method: Filter `post__in` array
- Catches: FiboSearch search results page
- Performance: Array operation, no DB queries

**Layer 3: WooCommerce Universal**
- Hook: `woocommerce_product_data_store_cpt_get_products_query`
- Method: Modify query args
- Catches: `wc_get_products()` (sliders, widgets)
- Performance: Query modification before execution

**Layer 4: Client-Side Fallback**
- Hook: DOM manipulation with jQuery
- Method: Hide elements after render
- Catches: FiboSearch dropdown only
- Reason: SHORTINIT mode bypasses PHP

### Caching Strategy

**User-Specific Cache** (30 minutes):
```
pam_hidden_products_{user_id}
pam_hidden_products_guest
```

**All Restricted Products** (30 minutes):
```
pam_all_restricted_products
```

**Cache Invalidation**:
- User login/logout
- Role changes (add/remove)
- Product updates
- Manual: `wp cache flush`
